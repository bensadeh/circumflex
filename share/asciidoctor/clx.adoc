== Name

clx - Hacker News in your terminal

== Synopsis

*clx* [_OPTION_]...

*clx* _COMMAND_ [_ARGS_]

== Description

*circumflex* lets you browse Hacker News in your terminal.
Visit the comment section or read the article in Reader Mode.

== Keybindings

_Enter_::
Read the comment section.

_Space_::
Read the article in Reader Mode.

_r_::
Re-fetch submissions for the current category.

_/_::
Search Hacker News.
See *Search*.

_Tab_::
Change category (use Shift + Tab to change in opposite direction).

_o_::
Open link to article in browser.
See *CLX_BROWSER*.

_c_::
Open comment section in browser.

_f_, _V_::
Add currently highlighted submission to favorites.

_x_::
Remove currently highlighted submission from favorites.

_u_::
Toggle read/unread for the currently highlighted submission.

_j_, _k_::
Move down/up one item.

_h_, _l_::
Previous/next page.

_g_, _G_::
Go to the top/bottom of the list.

_i_, _?_::
Bring up the help screen.

_q_, _Esc_::
Quit to prompt.

== Comment Section

The comment section has two interaction modes, toggled with _Tab_.

The default mode.

Scroll down/up one line.

_d_, _u_::
Scroll down/up half a page.

_Space_/_f_, _b_::
Page down/up.

_g_, _G_::
Go to top/bottom.

_n_, _N_::
Jump to next/previous top-level comment.

_/_::
Search within the comment section.
While a search is active, _n_/_N_ jump between matches.

_h_, _l_::
Collapse/expand one level.

_o_::
Open link to article in browser.

_c_::
Open comment section in browser.

_J_, _K_::
Open the comment section of the next/previous story on the front page.

Toggle collapse all.

_Tab_::

_i_, _?_::

Go back.

== Reader Mode

_j_, _k_::
Scroll down/up one line.

_d_, _u_::
Scroll down/up half a page.

_Space_/_f_, _b_::
Page down/up.

_g_, _G_::
Go to top/bottom.

_n_, _N_::
Jump to next/previous section.

_/_::
Search within the article.
While a search is active, _n_/_N_ jump between matches.

_h_, _l_::
Hide/show article images. See *--graphics* and *--show-images-on-open*.

_Tab_::
Open the link selector.
_j_/_k_ step between the article's links; _Enter_ follows the selected link in Reader Mode, and _q_ or _Esc_ walks back to the page it was opened from.
Links that cannot be read in Reader Mode are dimmed and inert.

_o_::

_c_::
Open comment section in browser.

_J_, _K_::
Open the next/previous story on the front page in Reader Mode.

_i_, _?_::
Bring up the help screen.

_q_, _Esc_::
Go back.

== Search

Press _/_ to search all of Hacker News, powered by Algolia.
Results open in their own tab in the header.

_s_::
Toggle sorting between popularity and date.

_d_::
Cycle the date range.

_r_::
Re-run the search with the current filters.

_q_, _Esc_::
Leave search.

== Options

All options can also be set persistently in _config.toml_ (see *FILES*); flags take precedence over the config file.

*-a* _n_, *--article-width*=_n_::
Set the article width in Reader Mode to _n_ columns.

*--categories*=_list_::
Set the categories in the header.
Available categories: top, best, new, ask, show, jobs, favorites.
Separate categories with comma.
The first category is the view shown on launch; it appears as the *clx* logo in the header rather than as a named tab.
Default: "top,best,ask,show,favorites".

*-c* _n_, *--comment-width*=_n_::
Set the comment width to _n_ columns.

*--graphics*=_mode_::
Set whether article images can be drawn at all.
Drawing them requires a terminal that supports the Kitty graphics protocol, such as Ghostty, Kitty or Konsole.
_auto_ (the default) asks the terminal and believes the answer; where none comes back, images are shown as a text placeholder and are not even downloaded.
_always_ assumes support without asking — for terminals that can draw images but whose answer never arrives, such as *tmux* without `allow-passthrough on`, or ones *clx* declines to probe because the question itself misbehaves there.
On a terminal that genuinely cannot draw images, forcing it on will print stray escape sequences.
_never_ refuses images outright: nothing is drawn, nothing is downloaded, and no escape sequence is sent.
Images are scaled to their size on the page and centered.
JPEG, PNG, GIF, WebP, AVIF and SVG images are drawn; other formats keep the placeholder.
Whether they are shown as soon as an article opens is set separately; see *--show-images-on-open*.

*--indent*=_n_::
Set the comment section indent size to _n_ spaces per nesting level.
When the requested indent would shrink a comment's text column below a readable minimum, the indent plateaus and deeper comments share an ancestor's indent level while the depth-colored bar continues to convey nesting.
Default: 1.

*-n, --nerdfonts*::
Enable Nerd Fonts icons as decorators for some elements (requires a Nerd Font-patched font).
Auto-enabled for Ghostty, which embeds Nerd Fonts.

*-d, --no-history*::
Disable marking stories as read.

*--pages*=_n_::
Set the number of pages to fetch per category, between 1 and 5.
Higher values fetch more stories but increase load time.
Only applies to top, new and best.
Ask and show always fetch one page because their story pools are small (~150-200 stories).
Default: 3.

*--show-images-on-open*::
Show images as soon as an article opens, rather than waiting for _l_.
Off by default, so articles open as text and images are revealed on demand.
Has no effect where images cannot be drawn.

*-v, --version*::
Show the current version of *circumflex*.

*-w* _value_, *--wide-view*=_value_::
Show the comment section and Reader Mode in a pane next to the front page instead of replacing it.
Set to _always_, _never_, or the minimum terminal width in columns at which the split kicks in.
Default: 180.

== Commands

*clx add* [_ID_]::
Add item to list of favorites by _ID_.

*clx article* [_ID_]::
Go directly to Reader Mode for a given item _ID_ without first going through the main view.

*clx comments* [_ID_]::
Go directly to the comment section for a given item _ID_ without first going through the main view.

*clx url* [_URL_]::
Open a URL directly in Reader Mode.

*clx clear*::
Clear the history of visited items.

*clx default-config*::
Generate a starter config file in the config directory.
See *FILES*.

*clx default-theme*::
Generate a starter theme file in the config directory.
See *FILES*.

== History

*circumflex* tracks which stories you have visited.
Visited stories are dimmed in the list.
When returning to a comment section, comments posted since your last visit are highlighted with a new-comment indicator.

Use *clx clear* to reset history, or *--no-history* to turn tracking off entirely.

== Favorites

Press _f_ to add the currently highlighted submission to your list of favorites.
Remove submissions from the Favorites page with _x_.

The favorites file is TOML, suitable for hand-editing or version control.
A _favorites.json_ from versions before 5.0 is migrated automatically and left in place.

== Environment

*XDG_CONFIG_HOME*, *XDG_CACHE_HOME*::
Override the directories where *circumflex* stores its config and cache files.
See *FILES* for what lives where.

*CLX_BROWSER*::
Command used to open URLs (article links and comment pages).
The URL is appended as the final argument.
For example, `CLX_BROWSER="firefox --new-tab"` opens links in a new Firefox tab.
When unset, the platform default opener is used (*open* on macOS, *xdg-open* on Linux, *rundll32* on Windows).

== Files

_$XDG_CONFIG_HOME/circumflex/config.toml_::
TOML configuration holding persistent defaults for the command-line options.
Run *clx default-config* to write a starter file listing every key with its default value.
Flags override values set here.

_$XDG_CONFIG_HOME/circumflex/theme.toml_::
TOML color theme.
Run *clx default-theme* to write a starter file pre-populated with every key and its default value.
Values may be named colors (`red`, `bright_blue`, ...), hex strings (`#ff8800`), or ANSI 256 indices (`0`-`255`).
Unrecognized values render with no styling.

_$XDG_CONFIG_HOME/circumflex/favorites.toml_::
Saved favorites.
A _favorites.json_ from versions before 5.0 is migrated here automatically.

_$XDG_CACHE_HOME/circumflex/history.json_::
IDs of visited stories.

When the XDG variables are unset, the platform defaults apply: `~/Library/Application Support/circumflex/` and `~/Library/Caches/circumflex/` on macOS; `~/.config/circumflex/` and `~/.cache/circumflex/` on Linux.
