== Name

clx - Hacker News in your terminal

== Synopsis

*clx* [_OPTION_]...

*clx* _COMMAND_ [_ARGS_]

== Description

*circumflex* lets you browse Hacker News in your terminal.
Visit the comment section or read the article in Reader Mode.

== Keybindings

_Enter_::
Read the comment section.

_Space_::
Read the article in Reader Mode.

_r_::
Re-fetch submissions for the current category.

_Tab_::
Change category (use Shift + Tab to change in opposite direction).

_o_::
Open link to article in browser.
See *CLX_BROWSER*.

_c_::
Open comment section in browser.

_f_, _V_::
Add currently highlighted submission to favorites.

_x_::
Remove currently highlighted submission from favorites.

_u_::
Toggle read/unread for the currently highlighted submission.

_j_, _k_::
Move down/up one item.

_h_, _l_::
Previous/next page.

_g_, _G_::
Go to the top/bottom of the list.

_i_, _?_::
Bring up the help screen.

_q_, _Esc_::
Quit to prompt.

== Comment Section

The comment section has two interaction modes, toggled with _Tab_.

=== Read Mode

The default mode.
Scroll through the comment section line by line.

_j_, _k_::
Scroll down/up one line.

_d_, _u_::
Scroll down/up half a page.

_Space_/_f_, _b_::
Page down/up.

_g_, _G_::
Go to top/bottom.

_n_, _N_::
Jump to next/previous top-level comment.

_h_, _l_::
Collapse/expand one level.

_o_::
Open link to article in browser.

_c_::
Open comment section in browser.

_Enter_::
Toggle collapse all.

_Tab_::
Switch to navigate mode.

_i_, _?_::
Bring up the help screen.

_q_, _Esc_::
Go back.

=== Navigate Mode

Jump between individual comments instead of scrolling line by line.

_j_, _k_::
Next/previous comment.

_d_, _u_::
Scroll down/up half a page.

_n_, _N_::
Jump to next/previous top-level comment.

_h_, _l_::
Collapse/expand current comment.

_o_::
Open link to article in browser.

_c_::
Open comment section in browser.

_Enter_::
Toggle collapse for current comment.

_Tab_::
Switch to read mode.

_i_, _?_::
Bring up the help screen.

_q_, _Esc_::
Go back.

== Options

*-a* _n_, *--article-width*=_n_::
Set the article width in Reader Mode to _n_ columns.

*--categories*=_list_::
Set the categories in the header.
Available categories: top, best, new, ask, show, favorites.
Separate categories with comma.
The first category is the view shown on launch; it appears as the *clx* logo in the header rather than as a named tab.
Default: "top,best,ask,show,favorites".

*-c* _n_, *--comment-width*=_n_::
Set the comment width to _n_ columns.

*-d, --disable-history*::
Disable marking stories as read.

*--indent*=_n_::
Set the comment section indent size to _n_ spaces per nesting level.
When the requested indent would shrink a comment's text column below a readable minimum, the indent plateaus and deeper comments share an ancestor's indent level while the depth-colored bar continues to convey nesting.
Default: 1.

*-n, --nerdfonts*::
Enable Nerd Fonts icons as decorators for some elements (requires a Nerd Font-patched font).
Auto-enabled for Ghostty, which embeds Nerd Fonts.

*--pages*=_n_::
Set the number of pages to fetch per category, between 1 and 5.
Higher values fetch more stories but increase load time.
Only applies to top, new and best.
Ask and show always fetch one page because their story pools are small (~150-200 stories).
Default: 3.

*-v, --version*::
Show the current version of *circumflex*.

== Commands

*clx add* [_ID_]::
Add item to list of favorites by _ID_.

*clx article* [_ID_]::
Go directly to Reader Mode for a given item _ID_ without first going through the main view.

*clx comments* [_ID_]::
Go directly to the comment section for a given item _ID_ without first going through the main view.

*clx url* [_URL_]::
Open a URL directly in Reader Mode.

*clx clear*::
Clear the history of visited items.

*clx default-theme*::
Generate a starter theme file in the config directory.
See *FILES*.

== History

*circumflex* tracks which stories you have visited.
Visited stories are dimmed in the list.
When returning to a comment section, comments posted since your last visit are highlighted with a new-comment indicator.

Use *clx clear* to reset history, or *--disable-history* to turn tracking off entirely.

== Favorites

Press _f_ to add the currently highlighted submission to your list of favorites.
Remove submissions from the Favorites page with _x_.

The favorites file is pretty-printed JSON, suitable for hand-editing or version control.

== Environment

*XDG_CONFIG_HOME*, *XDG_CACHE_HOME*::
Override the directories where *circumflex* stores its config and cache files.
See *FILES* for what lives where.

*CLX_BROWSER*::
Command used to open URLs (article links and comment pages).
The URL is appended as the final argument.
For example, `CLX_BROWSER="firefox --new-tab"` opens links in a new Firefox tab.
When unset, the platform default opener is used (*open* on macOS, *xdg-open* on Linux, *rundll32* on Windows).

== Files

_$XDG_CONFIG_HOME/circumflex/theme.toml_::
TOML color theme.
Run *clx default-theme* to write a starter file pre-populated with every key and its default value.
Values may be named colors (`red`, `bright_blue`, ...), hex strings (`#ff8800`), or ANSI 256 indices (`0`-`255`).
Unrecognized values render with no styling.

_$XDG_CONFIG_HOME/circumflex/favorites.json_::
Saved favorites.

_$XDG_CACHE_HOME/circumflex/history.json_::
IDs of visited stories.

When the XDG variables are unset, the platform defaults apply: `~/Library/Application Support/circumflex/` and `~/Library/Caches/circumflex/` on macOS; `~/.config/circumflex/` and `~/.cache/circumflex/` on Linux.
